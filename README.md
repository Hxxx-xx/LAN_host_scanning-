# 局域网主机扫描工具

跨平台局域网主机扫描工具，支持主机发现、端口扫描、OS指纹识别。

## 功能

- **主机发现**: 基于Ping检测，自动发现网段内存活主机
- **端口扫描**: 异步并发扫描常用端口 (22, 80, 443, 3389等)
- **OS识别**: 根据TTL指纹猜测目标操作系统
- **主机名解析**: 反向DNS解析获取主机名
- **跨平台**: 支持 Windows / Linux / macOS
- **多种输出**: 终端表格 + JSON/CSV 文件导出

## 安装

无需安装，Python 3.x 内置库即可运行。

## 使用方法

```bash
# 扫描当前网段
python lan_scanner.py

# 指定IP或网段
python lan_scanner.py --ip 192.168.1.0/24

# 指定单个IP
python lan_scanner.py --ip 192.168.1.100

# 跳过端口扫描(快速模式)
python lan_scanner.py --no-portscan

# 指定端口
python lan_scanner.py --ports 22,80,443,3389

# 输出到JSON文件
python lan_scanner.py -o result.json

# 输出到CSV文件
python lan_scanner.py -o result.csv
```

## 关于信息显示的说明

### 为什么主机名显示 "Unknown"？
主机名通过反向DNS解析获取。如果设备没有在DNS服务器注册（如家庭网络中的大多数设备），解析就会失败。这是正常现象，不代表设备不存在。

### 为什么MAC地址有时获取不到？
- macOS上如果ARP缓存中没有该IP的条目（设备刚联网且未通信过），就无法获取MAC地址
- 可以稍等几秒后重新扫描，让系统有更多时间学习ARP表

### 为什么有些设备识别不出类型？
设备类型识别基于端口特征匹配。如果设备：
- 没有开放任何常用端口（如部分手机、IoT设备）
- 使用非标准端口
- 被防火墙阻止了端口扫描

就无法根据端口识别类型，会显示为"❓ 未知设备"。

### 为什么扫描速度较慢？
ARP扫描需要逐个Ping网段内的254个IP地址，每个IP的超时时间约1秒。虽然只显示发现的设备，但扫描程序实际检测了所有254个IP。

## 技术细节

- 使用 Python asyncio 实现异步并发扫描
- 支持 Windows (ipconfig/ping) 和 Unix (ifconfig/ping) 平台适配
- 默认扫描端口: 21, 22, 23, 25, 53, 80, 443, 445, 3306, 3389 等

## 注意事项

1. 扫描速度受网络超时影响，整体约需3-5分钟完成
2. 部分网络设备可能禁用ICMP，导致无法被发现
3. 端口扫描可能触发防火墙警报
4. 请仅在授权网络中使用此工具
5. 扫描时建议关闭VPN，否则可能扫描到VPN分配的虚拟IP
